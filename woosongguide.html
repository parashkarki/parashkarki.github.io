RAG Flow

[Split] docs ‚Üí [Embed] vectors ‚Üí [Store] in DB ‚Üí [Search] top-k ‚Üí [Answer] using chunks
  • Chunk size: 400‚Äì800 chars with overlap.
  • Top-k: 3‚Äì5; adjust for speed/quality.

User Journey Map

Student arrives ‚Üí Reads tips ‚Üí Asks question ‚Üì Retrieve matching chunks (context) ‚Üì Compose grounded answer ‚Üì Show answer + "Sources" section

Working Structure of the App

  1. Input: text question
  2. Search: vector similarity on embeddings
  3. Compose: model responds using retrieved context
  4. Display: answer + sources + copy/print

Install & Commands (outside this portal)

Use locally to run the actual chatbot (Ollama + LlamaIndex + Gradio).

ollama pull phi3:mini
ollama run phi3:mini

gradio==4.44.0
llama-index==0.11.20
llama-index-embeddings-huggingface
llama-index-llms-ollama
chromadb

from llama_index.core import VectorStoreIndex, SimpleDirectoryReader, Settings
from llama_index.embeddings.huggingface import HuggingFaceEmbedding
from llama_index.llms.ollama import Ollama
import gradio as gr, os

os.makedirs("storage", exist_ok=True)
Settings.embed_model = HuggingFaceEmbedding(model_name="sentence-transformers/all-MiniLM-L6-v2")
Settings.llm = Ollama(model="phi3:mini", request_timeout=120)

docs = SimpleDirectoryReader("data").load_data()
index = VectorStoreIndex.from_documents(docs)
qa = index.as_query_engine(similarity_top_k=4, response_mode="compact")

def ask(q):
    try: return str(qa.query(q))
    except Exception as e: return f"Error: {e}"

demo = gr.Interface(ask, "text", "text", title="Woosong Guide AI")
if __name__ == "__main__":
    demo.launch()

Glossary

  • Model ‚Äî the ‚Äúbrain‚Äù that writes answers.
  • Embedding ‚Äî numbers representing meaning of text.
  • Vector DB ‚Äî fast database for embeddings.
  • RAG ‚Äî retrieve context first, then generate the answer.
  • Top-k ‚Äî how many chunks we retrieve per question.
  • Prompt ‚Äî instruction template to the model.
